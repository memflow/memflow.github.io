2. Compiling from Source

2.0. Prerequisites

memflow is written entirely in Rust. It is therefor required to have a Rust toolchain installed. memflow is verified to compile on the stable toolchain of Rust 1.45.1.

More information on how to install Rust can be obtained from the rustup project website.


2.1. Linux

memflow acquires physical memory via a so-called connector. By default, a tool written using memflow will use the plugin inventory system which will manage all available connectors internally.

To get started with the plugin inventory you need to have at least one connector installed. By default memflow will search all folders in $PATH for a sub-folder named memflow. All libraries the inventory finds in these sub-folders are then provided to the user.

To install one of the connectors simply clone the repository and run the installation script:

$ git clone https://github.com/memflow/memflow-coredump
$ cd memflow-coredump
$ ./install --system

When invoking the install script with the --system parameter
the connector is installed in the /usr/local/lib/memflow/ folder.
If no flag is specified the connector will be installed in the $HOME/.local/lib/memflow directory.
In case those paths are not in your $PATH variable please make sure to add them.

To install the command-line interface (CLI) simply clone the repository and compile it via cargo:

$ git clone https://github.com/memflow/memflow-cli
$ cd memflow-cli
$ cargo build --release --all-features

This builds the system daemon as well as the client interface for the cli.
The resulting binaries can now be found in the /target/release/ folder.
