2. Compiling from Source

2.0. Prerequisites

Memflow is written entirely in Rust. It is therefore required to have a Rust toolchain installed. Memflow is verified to compile on the stable toolchain of Rust 1.51.0.

More information on how to install Rust can be obtained from the rustup project website.


2.1. Connectors

Connectors play an integral role in the memflow ecosystem. They are responsible for providing the necessary functionality to communicate with a specific target. For example to connect to a qemu target the memflow-qemu connector is required.

2.1.1 memflowup

The recommended way to manage multiple connectors is through the memflowup utility. It is a small utility that allows rapid installation and updating of official connectors.

Installation

The recommended way to install memflowup is through the automated shell script:

$ curl --proto '=https' --tlsv1.2 -sSf https://sh.memflow.io | sh

Alternatively you can manually install it via cargo as well:

$ cargo install memflowup --force --version "=0.1.0-beta11"

Usage

A list of available commands can be obtained by running:

$ memflowup --help

To receive a guided installation of connectors run:

$ memflowup interactive

More detailed usage instructions can be found on the memflowup Github page

2.2. Cloudflow - A pluggable framework and UI for memflow

Cloudflow aims to be an extensible framework for memflow applications. The project is currently in its infancy, but it already has the following features:

  • FUSE interface
  • Full connector / OS chaining
  • Process information
  • Standalone minidump generator

Prerequisites

Cloudflow depends on the Rust FUSE package which depends on the FUSE development tools. Please check their repository for detailed installation instructions for your platform.

Installation

Currently cloudflow can only be built from source. This can be done using the following command:

$ cargo install cloudflow-node --git https://github.com/memflow/cloudflow

Usage

To run an elevated instance with FUSE use the following command:

$ cloudflow -ef

You should be able to see the following messages:

Mounting FUSE filesystem on /cloudflow
Initialized!

To create a new connector instance to the first QEMU VM instance on your system:

$ echo "my_qemu_vm qemu" >> /cloudflow/connector/new

Or alternatively connect to a specific VM e.g 'my-qemu-vm', you can pass an argument to the connector:

$ echo "my_qemu_vm qemu:my-qemu-vm" >> /cloudflow/connector/new

From here you can create a new OS instance on top of the QEMU connector like so:

$ echo "win -c my_qemu_vm win32" >> /cloudflow/os/new

Optionally you can specify arguments to this connector aswell, for example the architecture:

$ echo "win -c my_qemu_vm win32::arch=x64" >> /cloudflow/os/new

Full usage instructions can be found on the cloudflow Github page
